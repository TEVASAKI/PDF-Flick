## システム概要

### 目的

Androidのダウンロードフォルダに散乱したPDFファイルを、直感的なフリック操作で効率よく整理するアプリケーション。

### 対象ユーザー

- PDFファイルを頻繁に扱うビジネスユーザー
- ファイル整理を効率化したいユーザー
- シンプルで使いやすいUIを求めるユーザー

### 主要機能

1. **ダウンロードフォルダのスキャン**: PDFファイルを自動検出
2. **プレビュー表示**: PDFの最初のページを表示
3. **フリック操作**: 左右のフリックで「削除」「保存」を実行
4. **Undo機能**: 直前の操作を取り消し
5. **保存先設定**: ファイルの保存先フォルダを指定

## 機能仕様

### 1. ファイルスキャン機能

#### 概要
ダウンロードフォルダ内のPDFファイルを自動検出し、一覧表示します。

#### 仕様

| 項目 | 仕様 |
|------|------|
| スキャン対象 | ダウンロードフォルダ（`/Downloads/`） |
| ファイル形式 | `.pdf` |
| 検出順序 | ファイル名でソート（昇順） |
| 更新頻度 | アプリ起動時 |

#### 処理フロー

```
1. アプリ起動
2. ダウンロードフォルダをスキャン
3. PDFファイルを検出
4. ファイル情報を取得（名前、サイズ、更新日時）
5. 一覧表示
```

### 2. プレビュー表示機能

#### 概要
PDFの最初のページをプレビュー画像として表示します。

#### 仕様

| 項目 | 仕様 |
|------|------|
| 表示対象 | PDFの最初のページ |
| 画像形式 | PNG |
| 解像度 | 200 DPI |
| キャッシュ | メモリキャッシュ（最大50MB） |
| タイムアウト | 5秒 |

#### 実装方法

現在の実装：
- `react-native-pdf` を使用したレンダリング
- Base64 エンコードによるデータURI生成

将来の改善：
- Android ネイティブの `PdfRenderer` を使用
- より高速で効率的なプレビュー生成

### 3. フリック操作機能

#### 概要
ユーザーの指の動きを検出し、左右のフリックで「削除」「保存」を実行します。

#### 仕様

| 項目 | 仕様 |
|------|------|
| 検出方法 | PanResponder（React Native） |
| 判定閾値 | 50ピクセル以上の移動 |
| 右フリック | ファイルを保存フォルダに移動 |
| 左フリック | ファイルを削除（ゴミ箱に移動） |
| アニメーション | スプリングアニメーション（復帰） |

#### フリック判定ロジック

```typescript
if (Math.abs(dx) > SWIPE_THRESHOLD) {
  if (dx > 0) {
    // 右フリック: 保存
    handleKeep();
  } else {
    // 左フリック: 削除
    handleDelete();
  }
}
```

### 4. Undo機能

#### 概要
直前の操作を取り消し、ファイルを元の状態に戻します。

#### 仕様

| 項目 | 仕様 |
|------|------|
| 履歴サイズ | 最大50操作 |
| 取り消し対象 | 「保存」「削除」 |
| 復元方法 | ファイルの移動・復元 |
| 表示内容 | 処理済み件数（保存/削除） |

#### 操作履歴の構造

```typescript
interface HistoryEntry {
  action: 'keep' | 'delete';
  fileId: string;
  fileName: string;
  filePath: string;
  timestamp: number;
  metadata?: {
    destinationPath?: string;  // keep時の保存先
    trashPath?: string;        // delete時のゴミ箱パス
  };
}
```

### 5. 保存先設定機能

#### 概要
ユーザーが「保存」したPDFファイルを移動するフォルダを指定します。

#### 仕様

| 項目 | 仕様 |
|------|------|
| 設定方法 | ドキュメントピッカー（`expo-document-picker`） |
| 保存先 | ローカルストレージ（`pdf_flick_config.json`） |
| 変更可能 | いつでも変更可能 |
| デフォルト | 未設定 |

#### 設定ファイル形式

```json
{
  "saveFolderPath": "/path/to/save/folder"
}
```

interface PDFFile {

## API仕様

### ファイルシステムAPI（`usePDFFiles`）

#### `usePDFFiles()`

ダウンロードフォルダ内のPDFファイルを取得します。

**戻り値:**
```typescript
{
  files: PDFFile[];
  loading: boolean;
  error: string | null;
}
```

#### `moveFile(sourcePath: string, destinationFolder: string)`

ファイルを指定のフォルダに移動します。

**パラメータ:**
- `sourcePath`: 移動元ファイルパス
- `destinationFolder`: 移動先フォルダパス

**戻り値:**
```typescript
{
  success: boolean;
  path?: string;
  error?: string;
}
```

#### `deleteFile(filePath: string)`

ファイルを削除します。

**パラメータ:**
- `filePath`: 削除するファイルパス

**戻り値:**
```typescript
{
  success: boolean;
  error?: string;
}
```

#### `moveToTrash(filePath: string)`

ファイルをゴミ箱に移動します（復元可能）。

**パラメータ:**
- `filePath`: ゴミ箱に移動するファイルパス

**戻り値:**
```typescript
{
  success: boolean;
  trashPath?: string;
  error?: string;
}
```

#### `restoreFromTrash(trashPath: string, originalPath: string)`

ゴミ箱からファイルを復元します。

**パラメータ:**
- `trashPath`: ゴミ箱内のファイルパス
- `originalPath`: 復元先パス

**戻り値:**
```typescript
{
  success: boolean;
  error?: string;
}
```

### Undo機能API（`useUndoHistory`）

#### `addToHistory(entry: Omit<HistoryEntry, 'timestamp'>)`

操作を履歴に追加します。

#### `popLastEntry(): HistoryEntry | null`

直前の操作を取得して削除します。

#### `getStatistics()`

統計情報を取得します。

**戻り値:**
```typescript
{
  total: number;
  keep: number;
  delete: number;
}
```

## エラーハンドリング

### エラーの種類

| エラー | 原因 | 対応 |
|--------|------|------|
| ファイル取得エラー | ダウンロードフォルダにアクセスできない | ユーザーに権限許可を促す |
| ファイル移動エラー | 保存先フォルダが存在しない | フォルダを自動作成 |
| ファイル削除エラー | ファイルがロックされている | ユーザーに通知、再試行を促す |
| メモリ不足エラー | プレビュー画像がメモリに蓄積 | キャッシュをクリア |

### エラー表示

```typescript
Alert.alert(
  'エラー',
  'エラーメッセージ',
  [{ text: 'OK', onPress: () => {} }]
);
```

## パフォーマンス要件

### 応答時間

| 操作 | 目標時間 | 実装方法 |
|------|---------|---------|
| ファイル一覧表示 | 1秒以内 | 非同期処理 |
| プレビュー表示 | 500ms以内 | メモリキャッシング |
| フリック操作 | 2秒以内 | バックグラウンド処理 |
| Undo実行 | 500ms以内 | 同期処理 |

### メモリ使用量

| 状態 | 目標メモリ |
|------|-----------|
| 初期起動 | 50MB以下 |
| ファイル10個処理 | 100MB以下 |
| プレビューキャッシュ | 最大50MB |

### バッテリー消費

- バックグラウンド処理は最小化
- 定期的なスキャンは行わない
- ユーザー操作時のみ処理

## セキュリティ要件

### ファイルアクセス権限

- READ_EXTERNAL_STORAGE（ファイル読み取り）
- WRITE_EXTERNAL_STORAGE（ファイル書き込み）

### データ保護

- 設定ファイルはローカルストレージに保存
- ネットワーク通信なし（プライバシー保護）
- ファイルは暗号化されない（ユーザーの判断に委ねる）
