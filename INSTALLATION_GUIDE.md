## 前提条件

### ハードウェア要件

**開発環境**:
- CPU: Intel Core i5 以上（または同等のAMD）
- RAM: 8GB以上
- ストレージ: 5GB以上の空き容量

**本番環境（Android）**:
- OS: Android 8.0 以上
- RAM: 2GB以上
- ストレージ: 100MB以上の空き容量

### ソフトウェア要件

**開発環境**:
- Node.js 22.13.0 以上
- pnpm 10.4.1 以上
- Git 2.0 以上
- テキストエディタ（Visual Studio Code推奨）

**本番環境**:
- Android SDK（アプリビルド時）
- Gradle（アプリビルド時）

## 開発環境のセットアップ

### ステップ1: リポジトリのクローン

```bash
git clone <repository-url>
cd pdf_flick
```

### ステップ2: Node.jsとpnpmのインストール確認

Node.jsがインストール済みか確認してください。

```bash
node --version
# v22.13.0以上であることを確認

npm install -g pnpm
# または、既にインストール済みの場合
pnpm --version
# 10.4.1以上であることを確認
```

### ステップ3: 依存パッケージのインストール

```bash
pnpm install
```

このコマンドで、`package.json`に記載されているすべての依存パッケージがインストールされます。

**インストール時間**: 通常3～5分

### ステップ4: 開発サーバーの起動

```bash
pnpm dev
```

**出力例**:
```
  ➜  Local:   http://localhost:3000/
  ➜  Network: http://192.168.1.100:3000/
```

ブラウザで `http://localhost:3000` にアクセスしてください。

### ステップ5: 動作確認

ブラウザで以下が表示されることを確認してください。

- ページタイトル: 「PDF Flick」
- ヘッダー: 「PDF Flick」と説明文
- メインエリア: PDFプレビューカード
- ボタン: 「削除」「保存」「元に戻す」

## 本番環境への展開

### ステップ1: 本番ビルド

```bash
pnpm build
```

**出力ディレクトリ**: `dist/`

ビルド成果物には以下が含まれます。
- HTML、CSS、JavaScript（最小化済み）
- 静的アセット

**ビルド時間**: 通常1～2分

### ステップ2: ビルド成果物の確認

```bash
ls -la dist/
```

以下のファイル・ディレクトリが存在することを確認してください。

```
dist/
├── index.html
├── assets/
│   ├── index-*.js
│   └── index-*.css
└── public/
    └── favicon.ico
```

### ステップ3: プレビュー（オプション）

本番ビルドをローカルでプレビューできます。

```bash
pnpm preview
```

ブラウザで `http://localhost:4173` にアクセスしてください。

### ステップ4: サーバーへのデプロイ

ビルド成果物（`dist/`ディレクトリ）を本番サーバーにアップロードしてください。

**推奨ホスティング**:
- Manus（組み込みホスティング）
- Vercel
- Netlify
- AWS S3 + CloudFront

## 初期設定

### 設定ファイル

設定は以下のファイルで管理されます。

#### `client/src/index.css`

グローバルスタイル、カラーパレット、タイポグラフィを定義します。

**主な設定項目**:
- `--primary`: プライマリカラー（墨色）
- `--secondary`: セカンダリカラー（深い緑）
- `--accent`: アクセントカラー（薄い紅色）
- `--radius`: ボーダーラディウス

#### `vite.config.ts`

Viteビルドツールの設定ファイルです。

**主な設定項目**:
- ポート番号
- プロキシ設定
- ビルド出力ディレクトリ

#### `tailwind.config.ts`

Tailwind CSSの設定ファイルです。

**主な設定項目**:
- テーマカラー
- フォント設定
- プラグイン

### 環境変数

現在のバージョンでは環境変数の設定は不要です。

## トラブルシューティング

### 問題1: 開発サーバーが起動しない

**症状**: `pnpm dev` を実行してもサーバーが起動しない

**原因**: ポート3000が既に使用されている、またはNode.jsのバージョンが不適切

**対応**:
```bash
# ポート3000を使用しているプロセスを確認
lsof -i :3000

# プロセスを終了
kill -9 <PID>

# または、別のポートで起動
pnpm dev -- --port 3001
```

### 問題2: スタイルが反映されない

**症状**: CSSが読み込まれず、ページが崩れている

**原因**: Tailwind CSSのキャッシュが古い

**対応**:
```bash
# キャッシュをクリア
rm -rf .next
rm -rf node_modules/.vite

# 再度開発サーバーを起動
pnpm dev
```

### 問題3: PDFプレビューが表示されない

**症状**: プレビュー画像が読み込まれない、またはプレースホルダーが表示される

**原因**: PDFファイルが破損している、または非対応形式

**対応**:
- ファイルの整合性を確認
- 別のPDFファイルで試す
- ブラウザのコンソールでエラーメッセージを確認

### 問題4: フリック操作が反応しない

**症状**: フリック操作をしても何も起こらない

**原因**: タッチイベントが正しく認識されていない

**対応**:
- ブラウザの開発者ツールでタッチイベントをデバッグ
- スマートフォンのブラウザを再起動
- キャッシュをクリアして再度アクセス

### 問題5: ビルドが失敗する

**症状**: `pnpm build` 実行時にエラーが発生

**原因**: 依存パッケージが不足している、またはTypeScriptエラー

**対応**:
```bash
# 依存パッケージを再インストール
rm -rf node_modules
pnpm install

# TypeScriptの型チェック
pnpm check

# ビルド再実行
pnpm build
```
