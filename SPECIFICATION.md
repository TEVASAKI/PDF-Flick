## 概要

### アプリケーション名

PDF Flick

### 目的

Androidのダウンロードフォルダに散乱したPDFファイルを、直感的なフリック操作で効率よく整理するアプリケーション。

### 対応OS

- Android 8.0 以上

### ユーザー層

- PDFファイルを頻繁に扱うビジネスユーザー
- ダウンロードフォルダの整理に時間をかけたくないユーザー

## 機能仕様

### 1. PDFファイル一覧表示

**概要**: ダウンロードフォルダ内のPDFファイルを自動検出し、一覧として表示します。

**詳細**:
- ダウンロードフォルダをスキャンし、拡張子が `.pdf` のファイルを抽出
- ファイル名でソート（昇順）
- 隠しファイルは除外

**制限事項**:
- 1度に表示できるファイル数に上限なし（ただし、メモリ効率を考慮してページング実装を推奨）

### 2. PDFプレビュー表示

**概要**: 各PDFファイルの最初のページをプレビュー表示します。

**詳細**:
- PDFの最初のページを画像に変換（PNG形式）
- 解像度: 200 DPI（画面表示用）
- アスペクト比: 元のPDFに従う（通常は3:4）
- プレビュー生成に失敗した場合は、プレースホルダー画像を表示

**パフォーマンス**:
- プレビュー生成時間: 1ファイルあたり500ms以内
- キャッシング: メモリ内キャッシュで高速化

### 3. フリック操作（右フリック: 保存）

**概要**: PDFカードを右にフリックすると、ファイルを指定のフォルダに移動します。

**操作**:
- フリック距離: 50px以上で判定
- フリック速度: 制限なし（距離で判定）
- アニメーション: 300ms でカードがフェードアウト

**処理内容**:
1. ファイルを保存先フォルダにコピー
2. 元のファイルは削除
3. 次のPDFを表示
4. 操作履歴に「保存」を記録

### 4. フリック操作（左フリック: 削除）

**概要**: PDFカードを左にフリックすると、ファイルを削除します。

**操作**:
- フリック距離: 50px以上で判定
- フリック速度: 制限なし（距離で判定）
- アニメーション: 300ms でカードがフェードアウト

**処理内容**:
1. ファイルをゴミ箱に移動（または完全削除）
2. 次のPDFを表示
3. 操作履歴に「削除」を記録

**確認ダイアログ**: 削除時に確認ダイアログは表示しない（ワンアクションで完結）

### 5. Undo機能（元に戻す）

**概要**: 直前の操作（保存または削除）を取り消します。

**詳細**:
- 取り消し対象: 直前の1操作のみ
- 取り消し可能な操作数: 無制限（ただし、メモリ効率を考慮して実装）
- 復元方法:
  - 保存を取り消す場合: ファイルを保存先フォルダから元のダウンロードフォルダに戻す
  - 削除を取り消す場合: ファイルをゴミ箱から復元し、ダウンロードフォルダに戻す

**ボタン状態**:
- 操作履歴が空の場合: 「元に戻す」ボタンは無効化（グレーアウト）

### 6. 保存先フォルダ設定

**概要**: 初回起動時に、「保存」したPDFを移動するフォルダを指定します。

**詳細**:
- フォルダピッカーUIを表示
- ユーザーが任意のフォルダを選択可能
- 選択されたフォルダパスをローカルストレージに保存
- 2回目以降の起動時は、保存されたパスを使用

**デフォルト**: 指定されるまで、「保存」機能は無効化

## 画面仕様

### 1. 設定画面（初回起動時）

**レイアウト**:
- 画面中央に「フォルダを選択」ボタンを配置
- 選択されたフォルダパスを表示するテキストフィールド
- 「保存」ボタンで設定を完了

**フロー**:
1. アプリ起動 → 設定画面を表示
2. ユーザーが「フォルダを選択」ボタンをタップ
3. Androidのファイルピッカーが起動
4. ユーザーがフォルダを選択
5. 選択されたパスを表示
6. 「保存」ボタンをタップして設定完了
7. メイン画面に遷移

### 2. メイン画面（ファイル整理画面）

**レイアウト**:
- **ヘッダー**: アプリ名「PDF Flick」と説明文
- **メインエリア**: 
  - 中央: PDFプレビューカード（非対称レイアウト）
  - 左右: フリック方向を示すアイコン（薄い表示）
- **ボタンエリア**: 「削除」「保存」「元に戻す」ボタン
- **フッター**: 処理済みファイル数の表示

**表示要素**:
- PDFプレビュー画像（最初のページ）
- ファイル名
- 進捗表示（例: 「3 / 10」）

**デザイン**:
- 背景色: 白（`#FFFFFF`）
- テキスト色: 墨色（`#2C2C2C`）
- アクセント色: 深い緑（保存）、薄い紅色（削除）
- 余白を大きく取り、落ち着いた雰囲気を演出

### 3. 完了画面

**表示タイミング**: すべてのPDFを処理した後

**表示内容**:
- 「整理完了」というメッセージ
- 保存したファイル数
- 削除したファイル数
- 「もう一度整理する」ボタン

## 操作仕様

### フリック操作の詳細

| 操作 | 条件 | 結果 |
|------|------|------|
| 右フリック（50px以上） | ファイルが存在 | ファイルを保存フォルダに移動、次のファイルを表示 |
| 左フリック（50px以上） | ファイルが存在 | ファイルを削除、次のファイルを表示 |
| フリック距離 < 50px | - | キャンセル、カードは元の位置に戻る |

### ボタン操作

| ボタン | 状態 | 動作 |
|--------|------|------|
| 削除 | 常に有効 | 現在のファイルを削除、次のファイルを表示 |
| 保存 | 常に有効 | 現在のファイルを保存フォルダに移動、次のファイルを表示 |
| 元に戻す | 操作履歴あり時のみ有効 | 直前の操作を取り消す |

## データ仕様

### PDFファイル構造

```typescript
interface PDFFile {
  id: string;              // ファイルの一意識別子（UUID）
  name: string;            // ファイル名（拡張子含む）
  path: string;            // ファイルの完全パス
  preview: string;         // プレビュー画像（Base64またはURI）
  size: number;            // ファイルサイズ（バイト）
  modifiedDate: number;    // 最終更新日時（UNIXタイムスタンプ）
}
```

### 操作履歴構造

```typescript
interface HistoryEntry {
  action: 'keep' | 'delete';  // 実行された操作
  file: PDFFile;               // 対象ファイル
  timestamp: number;           // 操作実行時刻（UNIXタイムスタンプ）
}
```

### ローカルストレージ

| キー | 値の型 | 説明 |
|------|--------|------|
| `pdf_flick_save_folder` | string | 保存先フォルダのパス |
| `pdf_flick_history` | JSON | 操作履歴（配列） |

## エラーハンドリング

### エラーケース

| エラー | 原因 | 対応 |
|--------|------|------|
| PDFプレビュー生成失敗 | ファイル破損、非対応形式 | プレースホルダー画像を表示、ファイル名は表示 |
| ファイル移動失敗 | ストレージ容量不足、権限不足 | トースト通知でエラーメッセージを表示、操作は取り消し |
| ファイル削除失敗 | 権限不足、ファイルロック | トースト通知でエラーメッセージを表示、操作は取り消し |
| Undo失敗 | ファイルが既に削除されている | トースト通知でエラーメッセージを表示 |

### ユーザーへの通知

- **成功**: トースト通知（画面下部、2秒間表示）
- **エラー**: トースト通知（赤色、画面下部、3秒間表示）

## パフォーマンス要件

### 応答時間

| 操作 | 目標時間 |
|------|---------|
| ファイル一覧表示 | 1秒以内 |
| プレビュー表示 | 500ms以内 |
| フリック操作（ファイル移動） | 2秒以内 |
| Undo実行 | 500ms以内 |

### メモリ使用量

- 初期メモリ: 50MB以下
- ファイル10個処理時: 100MB以下
- プレビュー画像キャッシュ: 最大50MB

### ストレージ

- アプリ本体: 10MB以下
- キャッシュ（プレビュー画像）: 最大100MB

## セキュリティ

### ファイルアクセス権限

- **読み取り**: ダウンロードフォルダ内のPDFファイル
- **書き込み**: 指定された保存先フォルダ
- **削除**: ダウンロードフォルダ内のPDFファイル

### データ保護

- ローカルストレージに保存される情報（フォルダパス、操作履歴）は暗号化不要
- ユーザー個人情報は収集しない
